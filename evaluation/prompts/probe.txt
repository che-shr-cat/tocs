### Edge types
- **IMPORTS**: a Python `import` or `from X import Y` statement exists.
- **CALLS_API**: the file calls a public function exported by the target.
- **DATA_FLOWS_TO**: the output of this component is consumed by the target (e.g., return value passed as argument in an orchestrator).
- **REGISTRY_WIRES**: connected via a config file or registry, not a direct import.

**IMPORTANT**: Edge targets must be specific **file paths** (e.g., `stages/mod_a.py`), not directory paths (e.g., `stages/`). If a module connects to multiple files in a directory, list each file as a separate edge.

### Invariant structured types — field semantics

Each invariant has a `structured` object. Use the fields as follows for each type:

- **FORBIDDEN_EDGE**: A module must not depend on another.
  - `src`: the constrained module (specific file path), `dst`: the forbidden target (file path or null if "must not import any sibling"), `pattern`: short tag like `"no_import"`.
- **INTERFACE_ONLY**: Access must go through an interface/ABC.
  - `src`: null, `dst`: null, `via`: the interface provider file (e.g., `base.py`), `pattern`: null.
- **VALIDATION_CHAIN**: Data must flow through a required sequence. **Decompose chains into pairwise links** — one invariant per adjacent pair (e.g., mod_a→mod_b, mod_b→mod_c), not one holistic invariant for the whole chain.
  - `src`: first file in the pair, `dst`: second file in the pair, `via`: null, `pattern`: null.
- **NAMING_CONVENTION**: A structural naming rule (e.g., all stages match `mod_*.py`).
  - `src`: the directory or module, `dst`: null, `via`: null, `pattern`: the naming rule.
- **INVARIANT**: A runtime or behavioral constraint (e.g., "batch_size must be respected by all stages").
  - `src`: null, `dst`: null, `via`: null, `pattern`: short description of the constraint.
- **PURPOSE**: A design rationale (e.g., "each stage is a separate module for independent testing").
  - `src`: null, `dst`: null, `via`: null, `pattern`: short description of the rationale.

**IMPORTANT**: Use specific **file paths** in `src`, `dst`, `via` (e.g., `stages/mod_a.py`), not directory paths. Keep `pattern` values short (a few words), not full sentences or regexes.
